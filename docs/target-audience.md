# Target audience

Target audience

**Cloud Architect:**  
• Single platform to handle multiple and hybrid cloud services  
• Unified platform to manage both private and public cloud environments,
reducing operational complexity  
• Enable development teams to rapidly provision and deploy application
on any cloud, while maintaining security and architectural standards  
• Visibility and control over cloud consumption, so that I will have the
ability to optimize resource consumption across teams **DevOps
Engineer:**  
• Centralized governance for cloud resources, approvals and policies
across clouds  
• Easily configurable and customizable catalogs  
• Automated and streamlined process to reduce manual intervention and
errors  
• Track all actions and approvals based on historical data **Application
Developer:**  
• Provision infrastructure services through a single standard user
interface, without the need to log into each cloud provider  
• Quickly and easily configure a service based on standard templates, so
that I can focus better on developing my application rather than
infrastructure  
• View the estimated cost of the service I am creating, so that we can
better adapt to FinOps practices **IT Manager:**  
• View resource requests made by my team  
• Approve/reject the request based on technical requirements of the
resource requested.  
• Keep a tab on the cost of resource being provisioned.  
• Should be able to approve/reject the order based on the estimated cost
of the resource
